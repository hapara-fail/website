## ‚öñÔ∏è Decision Making Process

### Technical Decisions

1.  **Proposal**: Contributors can propose changes via GitHub Issues or Pull Requests.
2.  **Review**: The BDFL and designated community members review proposals for technical feasibility and alignment with the project mission.
3.  **Approval**: The BDFL has the final say on merging code or adopting new technologies.

### Major Changes

For significant changes to the project (e.g., new major features, architectural overhauls):

1.  **Discussion**: A detailed issue or proposal should be opened to solicit community feedback.
2.  **Consensus seeking**: While the BDFL has final authority, community input is heavily weighed to ensure changes benefit the users.
3.  **Decision**: The BDFL makes the final execution decision.
