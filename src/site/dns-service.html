Instructions

Method 1: Standard Settings (For personal or unmanaged Chromebooks)

  1. Click the time in the bottom-right, then click the Settings gear ⚙️.
  2. Go to Network > Wi-Fi, and click the > arrow next to your current network. Press the > arrow again.
  3. Expand the Network section.
  4. Under "Name servers," select Custom name servers.
  5. Enter 34.23.12.208 in all available fields.

Method 2: Network Internals (For when settings are blocked)

  1. Open a new tab and go to chrome://network.
  2. Select Network Select from the top navigation bar.
  3. Click "Render Network Select", then click on the + symbol directly below it or select another network.
  4. In the pop-up window, scroll down to the Name servers section.
  5. Change the setting from "Automatic name servers" to Custom name servers.
  6. Enter 34.23.12.208 in all available fields.

Recommended: Secure DNS (DNS-over-HTTPS)

Windows 10 Build 19628 and higher support DNS-over-HTTPS natively.

  1. Open Settings > Network & Internet > Status.
  2. Click Properties for your active connection.
  3. Scroll to DNS Settings and click Edit.
  4. Select Manual and enable IPv4.
  5. Preferred DNS: 34.23.12.208
  6. DNS over HTTPS: On (Automatic template) or use https://dns.hapara.fail/dns-query if manual input is required.

Standard IPv4 Setup

  1. Open Control Panel through Start menu or Windows search.
  2. Go to Network and Internet category and then to Network and Sharing Center.
  3. In the left panel, click "Change adapter settings".
  4. Right-click your active connection and select Properties.
  5. Find "Internet Protocol Version 4 (TCP/IPv4)" (or, for IPv6, "Internet Protocol Version 6 (TCP/IPv6)") in the list, select it and then click on Properties again.
  6. Choose "Use the following DNS server addresses" and enter 34.23.12.208.

Recommended: Secure DNS (Configuration Profile)

macOS Big Sur and higher support DNS-over-HTTPS natively via configuration profiles.

  1. Download the configuration profile.
  2. Your Mac should automatically open System Settings (or System Preferences) to the Profiles pane.
    If not, go to Apple menu > System Settings > Privacy & Security > Profiles.
  3. Double-click the profile in the list (it may be under a "Downloaded" or "Unsigned" section). Click Install... and follow the prompts to enter your Mac’s administrator password.

Standard IPv4 Setup

  1. Click the Apple icon and go to System Preferences.
  2. Click Network.
  3. Select the first connection in your list and click Advanced.
  4. Select the DNS tab and enter 34.23.12.208.

Recommended: Secure DNS (Configuration Profile)

iOS 14 and higher support DNS-over-TLS and DNS-over-HTTPS natively via configuration profiles.

  1. Download the configuration profile.
    Note: If using a third-party app (like Chrome), you may need to tap the file and select "Allow".
  2. A popup will appear saying "Profile Downloaded." Tap Close.
  3. Open Settings and tap Profile Downloaded (near the top) or go to General > VPN & Device Management.
  4. Select the downloaded profile, tap Install, enter your device passcode, and confirm until finished.

Standard IPv4 Setup

  1. From the home screen, tap Settings.
  2. Choose Wi-Fi in the left menu (it is impossible to configure DNS for mobile networks).
  3. Tap on the name of the currently active network.
  4. In the DNS field enter 34.23.12.208.

Recommended: Private DNS (Android 9+)

Android 9 supports DNS-over-TLS natively.

  1. Go to Settings > Network & internet > Advanced > Private DNS.
  2. Enter dns.hapara.fail.

Standard IPv4 Setup

  1. From the Android Menu home screen, tap Settings.
  2. Tap Wi-Fi on the menu. The screen listing all of the available networks will be shown (it is impossible to set custom DNS for mobile connection).
  3. Long press the network you're connected to, and tap Modify Network.
  4. On some devices, you may need to check the box for Advanced to see further settings. To adjust your Android DNS settings, you will need to switch the IP settings from DHCP to Static.
  5. Change DNS 1 to 34.23.12.208 and DNS 2 to 0.0.0.0.

Steps vary by distribution. These are general instructions for modern desktops using NetworkManager (Ubuntu, Fedora, etc.).

  1. Open Settings and navigate to the Network or Connections section.
  2. Find your active connection and click the gear icon ⚙️ to edit its settings.
  3. Go to the IPv4 tab.
  4. Disable the "Automatic" toggle next to DNS.
  5. In the DNS entry field, enter 34.23.12.208. Remove any other entries. Click Apply.
